# Карта мест свадьбы и маршруты — дизайн

Дата: 15 июля 2026 года  
Статус: согласованное направление, ожидает финальной проверки перед реализацией

## Цель

Сделать экран «Карта и адреса» полезным сразу после открытия: гость видит оба места свадьбы, автомобильный маршрут между ними и может одним нажатием построить маршрут от своего текущего местоположения до ЗАГСа или ресторана в Яндекс Картах.

## Пользовательский сценарий

1. Гость открывает экран «Карта и адреса».
2. Встроенная карта загружается автоматически, без промежуточной кнопки.
3. На карте видны две подписанные точки: Отделение ЗАГС Ленинского района и ресторан «Гоголь 15».
4. Между точками отображается автомобильный маршрут по дорогам.
5. Под картой находятся две самостоятельные ссылки-кнопки:
   - «Маршрут до ЗАГСа»;
   - «Маршрут до ресторана».
6. Ссылка открывает Яндекс Карты и предлагает маршрут от текущего местоположения гостя до выбранного места. На мобильном устройстве операционная система может передать ссылку установленному приложению Яндекс Карт.

## Архитектура

### Данные мест

Массив `wedding.locations` остаётся единственным источником названий, адресов и поисковых запросов. Маршрутные ссылки формируются отдельной чистой функцией, чтобы их можно было протестировать без браузерной карты.

### Встроенная карта

`YandexMap` загружает JavaScript API 2.1 один раз, создаёт карту при монтировании и уничтожает её при размонтировании. После геокодирования обоих адресов компонент:

- добавляет две метки с подписями и адресами;
- создаёт `ymaps.multiRouter.MultiRoute` с двумя координатами;
- использует автомобильный режим маршрутизации;
- включает автоматический подбор границ, чтобы весь маршрут был виден;
- оставляет только необходимые элементы управления масштабом и полноэкранным режимом.

Карта не запрашивает геолокацию посетителя: текущая позиция нужна только Яндекс Картам после перехода по внешней маршрутной ссылке.

### Экран локаций

Кнопка «Показать оба места на карте» и состояние скрытия удаляются. Карта всегда располагается под карточками адресов. Две маршрутные кнопки отображаются под картой независимо от состояния API.

Переключатель ЗАГС/банкет и карточки адресов сохраняются. Их существующие кнопки поиска заменяются общей понятной группой маршрутных действий, чтобы на экране не было конкурирующих ссылок с похожим смыслом.

## Ключ API

Ключ подключается через `VITE_YANDEX_MAPS_API_KEY`. Локальное рабочее значение хранится в игнорируемом `.env`, а `.env.example` остаётся без секретного значения. Поскольку JavaScript API работает в браузере, ключ виден посетителю; защита обеспечивается ограничением ключа по разрешённым доменам в кабинете разработчика Яндекса.

## Ошибки и резервное поведение

- До загрузки карты контейнер сохраняет размер, чтобы интерфейс не прыгал.
- При ошибке API выводится спокойное сообщение «Карта временно недоступна».
- Адреса и обе внешние маршрутные кнопки продолжают работать без встроенной карты.
- Если один адрес не удалось геокодировать, ошибка карты не ломает остальной экран.

## Адаптивность и доступность

- На десктопе маршрутные кнопки стоят в один ряд, на узких экранах — друг под другом.
- Высота карты ограничена адаптивным `clamp`, а экран остаётся прокручиваемым.
- Интерактивная карта получает понятное `aria-label`.
- Ссылки имеют минимум 44 × 44 пикселя, видимый фокус и открываются в новой вкладке с безопасным `rel`.
- Карта остаётся внутри `swiper-no-swiping`, чтобы её перетаскивание не переключало слайды.

## Проверка

1. Модульный тест формирования двух маршрутных URL.
2. Компонентный тест: карта отображается сразу, обе ссылки присутствуют и ведут в режим маршрута.
3. Браузерный тест экрана локаций с перехватом загрузки API либо проверкой резервного состояния.
4. Визуальная проверка на 390 × 844 и 1440 × 900.
5. Полный запуск существующих компонентных и E2E-тестов.

## Вне объёма

- Навигация по маршруту непосредственно внутри свадебного сайта.
- Запрос геолокации гостя самим сайтом.
- Выбор пешего или общественного транспорта на встроенной карте.
- Серверный Router API и отдельный платный ключ маршрутизации.
