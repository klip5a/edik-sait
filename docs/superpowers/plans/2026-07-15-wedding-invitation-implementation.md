# Свадебное приглашение — план реализации

Основание: `docs/superpowers/specs/2026-07-15-wedding-invitation-design.md`

## Правила выполнения

- Выполнять этапы последовательно; после каждого этапа запускать предусмотренные проверки.
- Не удалять существующую HTML/CSS/JS-версию до визуального подтверждения новой.
- Перед работой с библиотекой или API получать актуальную документацию через Context7.
- Перед каждым HTML/CSS/client-side JS этапом сверяться с `modern-web-guidance`.
- Не включать секреты и списки гостей в клиентскую сборку.
- Коммитить только завершённые и проверенные этапы.

## Этап 0. Зафиксировать исходную версию

Файлы: `index.html`, `css/**`, `js/**`, `images/**`.

Действия:

1. Проверить текущую страницу и отсутствующие/пустые файлы.
2. Зафиксировать существующие файлы отдельным baseline-коммитом.
3. Не перемещать и не удалять исходники: до миграции они служат контрольной версией.

Проверка: Git показывает чистое состояние, исходная версия открывается без необработанных JS-ошибок.
